# GBO2 Custom Parts Simulator - AI Handover Document

## 1. プロジェクト概要
本作は「機動戦士ガンダム バトルオペレーション2 (GBO2)」のカスタムパーツ最適化や火力・防御期待値を計算するためのWebベースのシミュレーターアプリです。
ユーザーは機体を選択し、装備するカスタムパーツを選ぶことで、残りスロットやHP・各種補正の変動をリアルタイムで確認できます。また、指定した「被弾配分」「攻撃配分」「優先ステータス（火力・耐久・機動）」に基づき、最適なパーツの組み合わせを自動計算（ブルートフォース/ヒューリスティック探索）する機能を備えています。

## 2. 技術スタック・アーキテクチャ
*   **フロントエンド**: HTML5, CSS3 (Vanilla), JavaScript (Vanilla, ES6+)
    *   外部ライブラリ非依存。DOM操作やイベントは全て `js/app.js` と `js/calculator.js` にて管理。
    *   UIデザインはダークテーマ、サイバーパンク/メカニカルな美学的アプローチを採用。
*   **バックエンド/データ基盤**: Python 3.10
    *   Wiki等からHTMLをリクエスト・スクレイピングするための強力なBot対策突破スクリプト群。
    *   取得したHTML群を解析（BeautifulSoupを使用）し、フロントエンドが利用可能な単一のJSON (`ms_data.json`) を生成する。

## 4. 実装済み機能（フロントエンド）
1.  **機体検索・フィルター**
    *   名前検索、兵科（汎用/強襲/支援）、コスト帯（200〜750）、機体LV に対応。
2.  **機体LV・強化施設（拡張スキル）の適用**
    *   機体ごとに用意された「強化段階（0〜6）」を選ぶと、HP増加や射撃・格闘補正の増加（上限突破）がベース値として乗算される。
3.  **カスタムパーツUIと状態保存**
    *   スロット（近・中・遠）の残量可視化。
    *   装備可能パーツのみを網掛け・装備不可はグレーアウト（`unequippable` クラス）制御。
    *   各パーツの「★」ボタンで所持/未所持切替。未所持パーツは `localStorage` に保存され、自動最適化対象から除外・UI非表示が可能。
4.  **計算・最適化エンジン (`GBO2Calculator`)**
    *   装備したパーツを加味した、「総合火力スコア」「有効HP」「加重平均カット率」の算出。
    *   **三すくみ補正**: 有利相手・不利相手を想定した際の火力値（1.3倍 / 0.8倍）と被ダメージ増減量（有効HPの計算）を算出表示。

## 5. 現在の仕掛り状況・次期AIへの最優先課題 (Critical Handover Tasks)
現在、以下の深刻なバグ・未完成機能（仕掛かり）が残っています。次期エージェントは読み込み次第、**真っ先にこれらの解決**に着手してください。

### 【至急】1. 兵科（汎用・強襲・支援）抽出の致命的バグ
*   **症状**: `tools/parse_all_ms.py` を実行した際、WikiのHTMLから機体カテゴリを抽出するロジック（`parse_ms_data.py`）が失敗し、**すべての機体（全563機）が「汎用」として出力されてしまう**深刻なバグが残っています。ウイングゼロEW等の強襲機も汎用となってしまいます。
*   **対策指示**: `data/ms_html_cache/` 内の実際のHTMLファイル（強襲機など）を直接 `grep_search` や `view_file` で調査し、「強襲」という文字列やアイコン画像の `alt` 属性などがHTMLのどこにどのように格納されているか見極め、判定ロジックを再構築してから再パースを実行してください。

### 【仕掛】2. カスタムパーツのデータ投入（DBの未完成）
*   **症状**: シミュレーターの計算ロジック自体は完成していますが、元となる「カスタムパーツのデータリスト（`data/custom_parts.json` やフロントエンドにハードコードされた配列など）」が不足しています。
*   **対策指示**: Wiki等から最新のカスタムパーツ（複合装甲、オーバーチューン系、サイコミュ系など）の「付与ステータス・消費スロット」を取り込み、ロジックが正常に適用されるようデータを追加充填してください。

### 【仕掛】3. 例外的な拡張スキル・割合（％）計算の対応
*   **症状**: 強化施設レベル選択による「拡張スキル」のうち、「機体HPが200増加」などの単純な数値加算は動作していますが、「サイコフレーム」等の一部割合増加（％）や、「旋回＋3」など特殊な補正の正規表現パースが漏れている場合があります。
*   **対策指示**: `calculator.js` 内の `applyEnhancements()` メソッドを見直し、未知の拡張スキルテキストにも対応できる柔軟な計算処理に変更してください。

### 4. UI/UXのさらなるブラッシュアップ
*   スマホ表示などレスポンシブデザインの微調整。
*   自動最適化ボタンのアニメーション（ローディング演出）の実装。

定期アップデート運用（月初・木曜の新機体追加）の際は、`batch_download_ms.py` を再実行してHTMLキャッシュを更新し、パースを実行する手順をとってください。
